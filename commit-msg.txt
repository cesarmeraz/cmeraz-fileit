feat: include EventId Name in structured logs (#53)

Every EventId in the codebase already had a human-readable Name (via nameof or explicit string), but the log output only ever showed the numeric Id. That made filtering and reading logs harder than it needed to be, since 'Id: 3011' is meaningless without looking up the event definition.

Two root causes fixed:

1. Log call sites were passing Events.Name.Id (just the int) instead of the full EventId struct. Microsoft.Extensions.Logging takes both overloads, but passing only the int discards the Name before Serilog ever sees it. Updated all 51 call sites across DataFlow, SimpleFlow, Services, and Infrastructure to pass the full EventId struct (Events.Name instead of Events.Name.Id) so the Name propagates into the log event properties.

2. Added an EventNameEnricher to the Serilog pipeline that pulls the Name out of the EventId StructureValue and exposes it as a top-level EventName property. This lets sinks (Console, OTLP, DatabaseSink) and dashboard filters search by name directly.

Console output template updated to emit both EventId and EventName fields. Structured log entries now show EventId: {Id: 3000, Name: 'DataFlowWatcher'} and EventName: 'DataFlowWatcher' alongside.

Also fixed a copy-paste bug in InfrastructureEvents.cs where FunctionStart's Name argument was nameof(FunctionEnd).

Verified end to end: dropped GLAccount.csv, confirmed DataFlow, Infrastructure, and Services events all show populated Name and EventName in Aspire's Structured log view.
